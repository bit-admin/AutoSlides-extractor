# AutoSlides Extractor

[**English Version**](README_EN.md) 👈 Click here for English

**AutoSlides Extractor** 是一款基于 Electron 和 Node.js 构建的跨平台桌面应用程序，旨在自动高效地从视频录像中提取演示幻灯片。它可以分析课堂录像、会议视频、网络研讨会和其他视频内容，捕获并保存幻灯片图像，大大提高笔记记录、内容归档和信息检索的效率。使用 AutoSlides Extractor，用户无需手动截图即可从冗长的视频中快速获取关键幻灯片图像。

该项目是从 [AutoSlides 项目](https://github.com/bit-admin/Yanhekt-AutoSlides) 中衍生出的一个独立工具。

## 功能

* **广泛的视频格式支持：** 支持主流视频格式，包括 MP4、AVI、MKV、MOV 和 WebM。无需额外转换，只需加载视频即可开始提取幻灯片。
* **精准的幻灯片切换检测：** 集成了多种幻灯片检测算法——用户可选择基础的像素差异方法或结合感知哈希和结构相似性（SSIM）的进阶模式来进行稳健的幻灯片切换检测。检测间隔（分析帧之间的时间步长）可自定义。
* **批量处理与队列：** 支持在一次会话中处理多个视频，将其添加到队列中。用户可一次性选择多个视频文件，应用会将它们排入队列并按顺序提取幻灯片，非常适合批量处理一系列讲座或会议。
* **可配置参数：** 用户可以调整幻灯片提取参数：
  * 帧间检测间隔（以秒为单位）。
  * 比较方法的选择（`basic` 或 `default` 算法）。
  * **双重验证**选项，要求幻灯片切换在多个帧中持续存在才能确认新幻灯片（减少因短暂变化导致的误报）。
  * **基于区域的检测**（高级）：可选地将幻灯片检测聚焦于视频帧的特定区域（适用于幻灯片仅占据屏幕一部分的情况）。通过指定区域大小和位置（例如右上角），应用将仅比较该部分的帧变化。
* **实时进度与预览：** 在提取过程中，应用会显示实时进度，包括已处理的帧数、检测到的幻灯片数量和已用时间。提取的幻灯片会在界面中实时预览，用户可随时停止处理。
* **后处理去除重复/不需要的幻灯片：** 应用可自动删除与某些“排除”幻灯片指纹匹配的图像。用户可标记特定幻灯片图像（或使用内置预设）以在未来提取中跳过。提取后，**后处理**步骤可将所有捕获的幻灯片与这些指纹进行比较，并删除过于相似的图像（例如重复的标题幻灯片、公司标志帧或其他重复内容）。
* **跨平台支持：** 为 **macOS** 和 **Windows** 提供原生安装程序，应用界面在各平台上保持一致。*(Linux 支持可能通过源码构建添加，但官方发布主要针对 macOS/Windows。)*
* **性能优化：** AutoSlides Extractor 使用多线程加速分析，利用多核 CPU 进行帧比较。视频解码和帧捕获由高效的 `ffmpeg` 引擎驱动。该工具能够在短时间内处理长视频（例如，在 Apple Silicon CPU 上，处理约 2.5 小时的视频，2 秒间隔，仅需约 1 分钟，基于内部测试）。

## 安装

### macOS

1. 从 [发布页面](https://github.com/bit-admin/AutoSlides-extractor/releases) 下载最新的 **AutoSlides Extractor** `.dmg` 安装程序。选择与您的 Mac 架构匹配的版本：Intel (`x64`) 或 Apple Silicon (`arm64`)。例如：**`AutoSlides Extractor-1.2.0-macOS-arm64.dmg`**。
2. 打开下载的 `.dmg` 文件，将 **AutoSlides Extractor.app** 拖入您的 **应用程序** 文件夹。
3. 首次启动时，macOS 可能会提示该应用来自未识别的开发者。如果遇到安全警告，打开终端并使用以下命令移除隔离属性：

   ```bash
   sudo xattr -d com.apple.quarantine /Applications/AutoSlides\ Extractor.app
   ```

   如有提示，输入您的密码。此步骤仅需执行一次以允许应用运行。
4. 从应用程序文件夹启动 **AutoSlides Extractor**。现在您可以正常使用该应用。

### Windows

1. 从 [发布页面](https://github.com/bit-admin/AutoSlides-extractor/releases) 下载最新的 Windows 安装程序 (`.exe`)。例如：**`AutoSlides Extractor-Setup-1.2.0-Windows-x64.exe`**（适用于 64 位 Windows）。
2. 运行安装程序并按照设置向导操作。您可以在安装过程中选择安装目录并选择是否创建桌面快捷方式。
3. 安装完成后，从开始菜单或桌面快捷方式启动 **AutoSlides Extractor**。应用将以直观的界面打开，供选择视频和配置选项。

*(无需额外依赖项——应用已捆绑了必要的 Node.js 运行时和 FFmpeg 二进制文件。在 macOS 上，FFmpeg 打包在应用内部；在 Windows 上，FFmpeg/FFprobe 可执行文件已包含在安装程序中。)*

## 使用方法

使用 AutoSlides Extractor 非常简单。按照以下步骤快速从视频中提取幻灯片：

1. **启动应用程序**。您将看到主窗口，包含选择视频和配置参数的选项。
2. **选择视频文件：** 点击 **“选择视频文件”**，选择要提取幻灯片的视频。您可以选择单个视频或**多个视频**。选择多个文件会将它们添加到处理**队列**中，队列列表在界面上可见。状态栏会显示队列中有多少个视频。如果仅选择一个视频，其路径将显示出来。
3. *(可选)* **选择输出目录：** 默认情况下，提取的幻灯片图像将保存到您的下载目录中的 `extracted` 文件夹中。如需使用其他位置，点击 **“选择输出目录”** 并选择一个文件夹。
4. *(可选)* **调整检测间隔：** 设置 **“检测间隔（秒）”** 以控制视频帧的采样频率。例如，2 秒的间隔意味着应用每 2 秒检查一次视频中的幻灯片变化（较小的间隔可能捕获短暂出现的幻灯片，但会增加处理时间）。
5. *(可选)* **选择比较方法：** 从 **“比较方法”** 下拉菜单中选择幻灯片切换检测算法：
   * **Default（默认）：** 使用感知哈希和 SSIM 的组合进行稳健检测（推荐用于大多数情况）。
   * **Basic（基础）：** 使用更简单的像素差异和变化比例方法（可能更快，但对噪声或微小变化敏感）。
6. *(可选)* **启用双重验证：** 如果勾选，应用将对幻灯片切换进行二次验证。这意味着当检测到幻灯片切换时，会等待确认变化是否在连续几帧中持续存在，然后才确认新幻灯片。启用此选项可以减少误报（例如忽略短暂的闪烁或光标移动，而非真正的幻灯片切换）。默认情况下此选项开启，要求变化在 **3 帧连续帧** 中持续存在才注册为新幻灯片。
7. *(可选)* **高级设置：** 点击 **“高级设置”** 调整额外参数：
   * **多核处理：** 启用或禁用多 CPU 核心的使用（默认启用以获得更好性能）。
   * **阈值：** 调整内部阈值，如 pHash 汉明距离、SSIM 相似度截止值、像素差异容差等（仅限高级用户——默认值已针对典型幻灯片内容进行调优）。
   * **帧视频质量：** 您可以降低提取的帧图像质量（1 = 最高质量，31 = 最低质量）以加速处理或节省磁盘空间。默认值为 1（无质量损失）。
   * **感兴趣区域：** 如果幻灯片占据视频的特定区域（例如在录制的会议中包含摄像头画面和幻灯片），您可以启用基于区域的比较并指定区域尺寸和位置。应用将仅分析每帧的该部分是否有变化。
   * **后处理选项：** 管理用于过滤不需要幻灯片的指纹。在此可以查看、添加或删除 **排除指纹**（详见下文 *后处理*）。
8. **开始提取：** 点击 **“开始提取”** 开始处理。如果您将多个视频添加到队列中，按钮将显示 **“开始队列”**，视频将自动逐一处理。处理过程中，进度条和状态文本会实时更新，界面将显示已处理的总帧数和提取的幻灯片数量。
9. **查看结果：** 检测到幻灯片后，其缩略图会出现在窗口底部的预览区域中。所有提取的幻灯片将以 PNG 图像文件格式保存在您选择的输出目录（或默认的 `~/Downloads/extracted` 文件夹）中。您可以从文件资源管理器中打开该文件夹查看全尺寸图像。每张幻灯片图像的文件名中都带有时间戳（例如 `slide_001_timestamp.png`）。
10. *(可选)* **停止或重置：** 您可以随时点击 **“停止提取”** 暂停处理。如果正在处理队列，此操作将在完成当前视频后停止。要清除当前视频/队列并重置界面（清除预览和进度），点击 **“重置”**。如果队列处于活动状态，重置也将清除队列。
11. *(可选)* **后处理幻灯片：** 提取完成后，如果您配置了任何 *排除指纹*（见下一部分），可以运行后处理步骤以删除与这些指纹匹配的幻灯片。点击 **“选择幻灯片目录”** 并选择幻灯片图像文件夹（输出目录），然后点击 **“运行后处理”**。应用会将每张幻灯片图像与存储的指纹进行比较，并删除高度相似的图像（例如重复的封面幻灯片）。将显示删除了多少图像的摘要。

## 后处理与指纹排除

AutoSlides Extractor 提供了一个强大的 **排除列表** 功能，可以自动过滤掉您认为不重要或重复的幻灯片。该功能通过使用 **图像指纹**（基于 SSIM）来识别相似图像：

* 在 **高级设置** 中，您可以通过点击 **“添加排除图像”** 来添加新指纹（这会打开文件选择器以选择图像，例如要排除的幻灯片截图）。然后系统会提示您配置该图像的指纹区域（如果有），之后应用会计算并存储该图像的指纹。
* 排除指纹列表在高级设置中可见。每个指纹都有一个 ID、名称和相似度阈值。您可以编辑或删除列表中的指纹。某些指纹可能是内置 **预设**（在列表中标记为预设），涵盖常见情况。
* 当对幻灯片目录运行 **后处理** 时，应用会加载每个存储的指纹（以及任何预设），并使用 SSIM 将文件夹中的每张幻灯片图像与这些指纹进行比较。如果幻灯片图像与任何排除指纹的 **相似度 >= 95%**（SSIM ≥ 阈值，默认 0.95），该幻灯片图像文件将被删除。这有助于自动剔除检测到但不需要的幻灯片（例如系列视频中重复的介绍幻灯片或公司标志帧）。
* 指纹比较会考虑为指纹定义的指定区域。这意味着您可以针对幻灯片的一部分进行操作（例如，忽略仅因发言者摄像头缩略图不同的幻灯片，专注于幻灯片内容区域）。

此排除机制是可选的；如果未配置排除指纹，则不需要后处理步骤。指纹配置保存在应用的用戶数据中（因此您的排除列表会在运行之间持久保存）。

## 项目结构

AutoSlides Extractor 项目的组织结构分为几个关键目录和文件：

* **`src/main/`** – **主进程代码 (Node.js)**。包括：
  * `main.js` – Electron 应用的入口点（在主进程中运行）。它创建应用窗口、设置 IPC 处理程序、加载和保存配置、通过 ffmpeg 协调视频帧提取，并管理幻灯片检测逻辑（包括为分析生成工作线程）。
  * `preload.js` – 渲染器的预加载脚本。它定义了一个安全桥接（`contextBridge`），为渲染器暴露某些 API 用于 IPC（例如，`electronAPI` 方法，如 `selectVideoFile`、`getConfig`、`extractFrames` 等）。
  * **`utils/`** – 主进程的辅助模块。值得注意的是，`utils/image-processor.js` 实现了图像处理功能（例如调整大小、灰度转换、简单图像解码），用于幻灯片比较。*(这是使用重量级库如 Sharp 的轻量级替代方案。)*
  * **`workers/`** – 用于并行图像分析的工作线程脚本。例如，`workers/image-processor.js` 是每个工作线程中运行的代码，用于比较帧并检测并行块中的幻灯片变化。主进程默认生成多个工作线程（最多比 CPU 核心数少一个）以加速处理。
* **`src/renderer/`** – **渲染进程代码（前端）**。包含应用用户界面的文件：
  * `index.html` – 应用窗口的 HTML 结构，定义按钮、输入、预览区域、高级设置模态框等的布局。
  * `renderer.js` – 处理用户交互并更新 UI 的前端 JavaScript。它调用暴露的 `electronAPI` 方法来调用后端操作（例如打开文件对话框、开始提取、接收进度事件等）。它还管理队列 UI、高级设置模态框逻辑以及进度和状态的实时更新。
  * `styles.css` – 用于样式化应用 UI 的 CSS 文件。
* **`src/assets/`** – **静态资产和预设**。包含随应用打包的图像和其他资产。值得注意的是，它包括一个 `fp/` 子文件夹，内有预设指纹文件和一个 `presets.json` 配置文件。这些预设可能包括常见幻灯片元素的示例指纹（如果提供），并在首次运行时加载以填充排除列表。`assets` 文件夹通常还包含应用图标和运行时所需的任何图形资产。
* **`build/`** – 打包的构建配置和资源：
  * 不同平台的图标（macOS 的 `icon.icns`，Windows 的 `icon.ico`）、安装程序背景图像，以及可能的特定平台二进制文件。对于 Windows，`build/binaries/win-x64/` 文件夹包含应用使用的 `ffmpeg.exe` 和 `ffprobe.exe`（这些文件在安装应用时被复制）。在 macOS 上，ffmpeg 通过 `ffmpeg-static` npm 包包含，并在构建时解压。
  * Electron Builder 配置在 `package.json` 的 `build` 字段中定义，指定如何为 Mac（dmg）和 Windows（NSIS 安装程序）打包应用。
* **配置文件和数据文件：** 应用会创建一个用户特定的数据目录（通常在操作系统的应用数据路径中）来存储 `config.json`（记住您上次使用的设置，如输出目录、间隔、算法选择等）和一个 `fingerprints` 目录用于存储指纹数据。这确保您的偏好在会话之间和应用更新后持久保存。

## 技术概述

在内部，AutoSlides Extractor 遵循经典的 **Electron 架构**，前端 **渲染进程** 和后端 **主进程** 分离：

* **渲染进程**（前端）负责图形界面——处理用户输入并显示输出。它通过 IPC（进程间通信）调用与主进程通信，这些调用通过预加载脚本中的安全 `electronAPI` 桥接暴露。例如，当用户点击“开始”时，渲染器调用一个方法，在主进程中触发帧提取，并监听进度事件以更新 UI。
* **主进程**（后端）负责繁重的工作：
  * 它使用 **Fluent FFmpeg**（`fluent-ffmpeg` 库）打开视频文件并以指定间隔提取帧。帧默认保存为图像文件（JPEG）到临时目录中。
  * 在提取帧时，主进程要么按顺序处理它们，要么将它们分布到 **工作线程**（使用 Node.js **Worker Threads**）进行分析。默认情况下，多核分析是启用的，应用会根据您的 CPU 核心数量生成多个工作线程（例如，如果您有 4 个核心，可能会使用 3 个工作线程，留一个核心给主线程）。
  * 每个工作线程在一组帧上运行幻灯片检测算法。算法将每帧与前一帧进行比较，以决定是否发生了幻灯片切换。检测到的幻灯片帧被送回主进程。
  * 主进程聚合工作线程的结果，更新进度，并在确认新幻灯片时，将幻灯片图像保存到输出目录并通知渲染器（渲染器随后在 UI 中更新预览）。
  * 主进程还处理保存配置（例如当您更改设置时）和在启动时加载配置，以及管理排除指纹存储和后处理程序。

### 幻灯片切换检测算法

AutoSlides Extractor 的核心逻辑是确定视频中何时发生幻灯片切换。它提供了两种比较模式：

**1. 基础模式 (`basic`)：** 一种快速算法，对连续帧进行逐像素比较。它计算每个像素的绝对差异，并统计颜色值超出某个阈值的像素差异数量。如果差异像素的比例超过设定比例（默认 0.5% 的像素差异值大于 30），则标记为显著变化。此方法简单，适用于较大变化，但对噪声敏感（小的移动或闪烁可能会触发）。

**2. 默认模式 (`default`)：** 一种更稳健的算法，结合 **感知哈希 (pHash)** 和 **结构相似性指数 (SSIM)** 来判断幻灯片是否已更改：
* **感知哈希 (pHash)：** 将帧缩小并进行变换（DCT）以生成指纹哈希。通过比较两帧的哈希，得到汉明距离——实质上是图像差异的度量。如果汉明距离低于阈值（帧非常相似，例如阈值 ≤ 5），则认为内容相同。
* **SSIM (结构相似性)：** 该指标以人类视觉的方式评估图像，关注亮度、对比度和结构信息。SSIM 产生一个介于 -1 和 1 之间的相似度分数；两张相同的图像 SSIM = 1。如果两帧之间的 SSIM 高于高阈值（例如 ≥ 0.999，表示几乎没有差异），则认为它们相同。
* **综合决策：** 在默认模式下，只有当帧在两种相似性检查中都失败时——即 pHash 差异较大 *或* SSIM 低于阈值——才会检测到新幻灯片。这种组合有助于捕捉涉及显著视觉差异的幻灯片切换（pHash 会捕捉到）以及更微妙的变化（即使 pHash 有时未捕捉到小的变化，SSIM 也会下降）。

**双重验证：** 无论使用哪种模式，如果启用“双重验证”机制，当发现一帧足够不同以表明可能是新幻灯片时，算法不会立即确认。它会临时将此帧标记为 *候选幻灯片*，然后要求接下来的几帧（默认 2 帧）与此候选帧保持相似，以确认幻灯片确实发生了变化，而视频并非只是闪烁或显示短暂覆盖。如果接下来的帧确认了变化，候选帧最终被确认为新幻灯片；如果没有，则可能是误报，幻灯片计数不会增加。这种方法大大减少了因光标移动、小过渡动画或短暂字幕出现等导致的误报。

**文件大小启发式：** 作为初步快速检查，应用还比较连续帧图像的文件大小。如果大小差异超过一定百分比（默认 5%），则强烈表明图像不同（因此立即视为变化）。相反，如果文件大小几乎相同（差异低于 0.05%），则假定帧相同并跳过更昂贵的计算。这种启发式方法通过避免对明显相同的帧进行不必要的像素或哈希计算来加速处理。

所有这些阈值和常量（如 pHash 汉明距离限制、SSIM 阈值、像素差异比例等）都可以通过高级设置或修改源代码默认值进行配置。默认值已针对典型场景中的幻灯片检测进行了优化。

### 性能与并行性

幻灯片提取是一项 CPU 密集型任务，尤其是图像比较步骤。AutoSlides Extractor 设计为利用多核 CPU，通过分割工作来提升性能。默认情况下，它会生成工作线程以并行处理帧序列——实质上将视频分成帧块供每个工作线程处理。这可以在多核系统上实现接近线性的加速，利用现代处理器更快地分析视频。您可以在设置中禁用多核处理（如果您希望为其他任务保留 CPU），但通常建议保持启用以获得最佳性能。

内存方面，应用每个工作线程一次处理一帧（帧被加载、比较，然后释放），因此可以处理长视频而无需一次性将整个视频加载到内存中。然而，对于极高分辨率的视频或非常短的间隔，帧数量可能很大，因此请确保输出目录有足够的磁盘空间用于提取的图像。

### 高级功能：基于区域的分析与指纹识别

如前所述，应用支持将幻灯片切换检测聚焦于视频帧的子区域。这对于视频中包含画中画的情况很有用，例如幻灯片仅占据帧的一部分。在这种情况下，幻灯片外的随机变化（如发言者视频或背景）可能会干扰检测。通过启用基于区域的比较，您可以指定一个区域（宽度、高度和对齐方式，如中心、右上等），算法将仅比较每帧的该区域。区域对齐选项包括上/中/下和左/中/右，因此您可以轻松地定位，例如帧的右下四分之一如果那是幻灯片所在位置。基于区域的比较也应用于 SSIM 指纹计算中——确保如果您设置了区域，指纹和比较仅考虑图像的该部分。

用于后处理的指纹识别系统也可扩展。应用附带一些 **预设指纹**（位于 `src/assets/fp` 中），这些指纹可能对应于常被排除的幻灯片或模式。这些指纹在首次运行时自动初始化并存储在用户的指纹存储中。指纹文件（`.fp` 扩展名）和 `presets.json` 定义了这些预设，每个预设都有自己的阈值和可选的区域配置。开发者可以根据需要添加新的预设指纹，或者用户可以通过 UI 导入/导出指纹（例如，与同事共享排除列表）。

在底层，指纹实质上是一个小的二进制文件，代表图像的基于 SSIM 的签名（带有可选区域）。比较两个指纹会得出一个相似度分数（0 到 1）。如果相似度高于配置的阈值，则认为图像匹配。

总体而言，这些高级功能（区域聚焦和指纹排除）使工具适应各种现实场景，确保以最小的噪声尽可能干净地提取幻灯片。

## 开发指南

如果您是开发者，有兴趣探索或扩展 AutoSlides Extractor，本节将帮助您开始了解代码库。

**前提条件：** 您应安装 [Node.js](https://nodejs.org/)（项目已针对与 Electron 35.x 对应的 Node.js 和 Electron 版本进行测试）。确保您也安装了 npm（Node 包管理器）。

**设置和开发运行：**

1. **克隆仓库：** 将 GitHub 仓库 `bit-admin/AutoSlides-extractor` 克隆到您的本地机器。
2. **安装依赖项：** 在项目目录中运行 `npm install`。这将安装 Electron、Electron-Builder 以及所有 Node 依赖项（包括 `fluent-ffmpeg`、`ffmpeg-static` 等，如 `package.json` 中所列）。`ffmpeg-static` 包将为开发使用提供 FFmpeg/FFprobe 二进制文件。
3. **以开发模式启动应用：** 运行 `npm run dev`。这将以开发模式启动 Electron 应用（如果配置了热重载）。主窗口应出现，您可以打开开发者工具（Ctrl+Shift+I 或 Cmd+Opt+I）以调试渲染进程。在开发模式下，应用将使用 `ffmpeg-static` 提供的 `ffmpeg` 和动态代码（未打包）。
4. **文件夹结构：** 请参考上文的 *项目结构* 部分以了解关键文件概览。进行更改时：
   * 前端更改（HTML/JS/CSS）将在重新加载应用窗口时反映。
   * 主进程更改需要重启 Electron 进程。利用终端中的控制台日志记录主进程日志，以及浏览器开发者工具控制台记录渲染器日志。
5. **构建包：** 要创建可分发的安装程序，运行 `npm run build`。构建过程使用 Electron Builder，默认情况下会在 `dist/` 目录中生成输出。如果您想构建 Linux 包或更改应用元数据，请确保在 `package.json` 的 `build` 字段中调整配置。默认情况下：
   * 在 macOS 上，将创建 `.dmg` 安装程序（和 `.app` 包）并包含应用名称和版本。
   * 在 Windows 上，将创建 NSIS 安装程序 `.exe`（和支持文件）。
   * 构建过程配置为嵌入必要的二进制文件和资源，如前所述（参见 `package.json` 中的 `extraResources` 和 `asarUnpack` 设置）。
6. **开发期间的配置和数据：** 应用将在 Electron 用户数据文件夹中读写配置和指纹数据。在开发模式下，这通常是类似 `~/Library/Application Support/AutoSlides Extractor (development)`（macOS）或 `%APPDATA%/AutoSlides Extractor (development)`（Windows）的文件夹。您可以在应用未运行时删除其中的 `config.json` 来重置配置。

**扩展功能：**

* *添加新的检测算法：* 如果您想实现新的幻灯片切换检测方法，您可能需要：
  * 修改 `src/main/workers/image-processor.js` 中的比较逻辑。函数 `compareImages(buffer1, buffer2, method)` 是当前实现 `basic` 和 `default` 方法的地方。您可以添加一个新的 `method` 字符串并在那里实现您的算法。还需更新 `index.html`/`renderer.js` 中的 UI 下拉菜单以允许选择您的方法。
  * 确保您的方法的阈值或参数（如需要）是可配置的（您可以扩展配置 JSON 结构和高级设置模态框以添加新字段）。
* *修改 UI/UX：* UI 使用纯 HTML/CSS/JS 构建，因此您可以修改 `index.html` 和 `styles.css` 以进行布局或样式更改。要添加新的 UI 控件，还需更新 `renderer.js` 以处理其行为，并根据需要与主进程 IPC 调用链接。
* *使用 IPC：* 渲染器和主进程之间的通信通过 `electronAPI` 完成（参见 `preload.js`）。如果您添加需要主进程工作的新功能，请定义一个 IPC 通道（在 `main.js` 中使用 `ipcMain.handle` 处理请求，或使用 `ipcMain.on`/`ipcRenderer.send` 处理事件）。在 `preload.js` 中暴露一个函数，让渲染器调用它。代码中有许多示例（例如，`main` 中的 `ipcMain.handle('extract-frames', ...)` 和 `preload` 中的 `electronAPI.extractFrames`）。
* *测试：* 当前，项目不包含自动化测试。作为开发者，您可以通过应用运行不同视频来测试更改。使用已知幻灯片切换的样本测试视频（甚至是合成视频）有助于验证检测准确性和性能。
